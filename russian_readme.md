## ☁️ Обзор

**Fire CLI** — это ультралегкая утилита, написанная на **Rust**, которая превращает ваш терминал в уютный камин. Благодаря прямой работе с буфером и отсутствию тяжелых зависимостей, анимация остается плавной даже на самом слабом железе.

### Почему это круто?
* **Кроссплатформенность**: Работает везде, где есть поддержка Rust и `libc`. Будь то Windows, macOS, дистрибутивы Linux или BSD.
* **Чистая архитектура**: Весь визуал строится на стандартных потоках вывода, гарантируя работу в минималистичных окружениях.

| Возможность | Описание |

> [!TIP]
> **ASCII-движок**: Визуализация огня основана на ANSI-последовательностях. Для идеального градиента ваш терминал должен поддерживать 24-битный цвет (TrueColor).

## 💻 Совместимость с терминалами

На устаревших системах (Windows 10 и ниже) стандартные консоли `cmd.exe` или `PowerShell.exe` часто не справляются с интенсивным динамическим выводом. Это приводит к следующим проблемам:

* ⚠️ **Артефакты**: Появление «мусорных» управляющих символов вроде `←[0K` поверх анимации.
* 📉 **«Дёрганая» картинка**: Низкая скорость обработки потока данных вызывает разрывы (tearing) и мерцание кадров.
* 🚫 **Проблемы с TrueColor**: Старые консоли не поддерживают 24-битный цвет.

**Для безупречной картинки подойдёт любой современный терминал с поддержкой VT-последовательностей:**

* 🎨 **Рекомендованные:** [**Windows Terminal**](https://aka.ms/terminal), [**Alacritty**](https://alacritty.org/) или [**WezTerm**](https://wezfurlong.org/wezterm/).
* 🚀 **Также подойдут:** **Kitty**, **Foot**, **Konsole** и любой другой эмулятор с поддержкой 24-битного цвета.
* 🛠️ **Главное условие:** Терминал должен корректно обрабатывать управляющие коды.

> [!TIP]
> Если вы видите странные символы `←[0K` или анимация «заикается» — это знак того, что ваш текущий терминал устарел. Время обновиться!

## 🛠️ Сборка и установка

> ❕ Для компиляции и запуска **Fire CLI** в вашей системе должен быть установлен актуальный стек [**Rust**](https://www.rust-lang.org/tools/install) (cargo, rustc).

### 1. Сборка из исходников

Склонируйте репозиторий и соберите оптимизированный бинарный файл:

```bash
# Клонируем репозиторий
git clone https://github.com/horizonwiki/fire
cd fire

# Собираем финальную версию
cargo build --release
```

> ❕ После завершения сборки исполняемый файл будет находиться в директории `/target/release/`

### 2. Установка в систему

#### 🐧 Linux и macOS

Скопируйте скомпилированный файл в одну из стандартных системных директорий:

```bash
sudo cp target/release/fire /usr/local/bin/
```

#### 🪟 Windows

1. Создайте папку для утилит (например, `C:\bin`) и переместите туда `fire.exe` из `target\release\`.
2. Нажмите `Win + R`, введите `sysdm.cpl`, перейдите в **Дополнительно → Переменные среды**.
3. В разделе «Переменные среды пользователя» найдите `Path`, нажмите «Изменить» и добавьте путь к папке (например, `C:\bin`).
4. Перезапустите терминал, чтобы изменения вступили в силу.

> [!TIP]
> Если вы используете пакетный менеджер `winget`, в будущем утилита будет доступна для установки одной командой!

## 🎮 Использование и управление

Запустите анимацию командой `fire`. Для настройки используйте следующие флаги:

### ⚙️ Опции запуска

| Флаг | Полное имя | Описание | Значения |
| :--- | :--- | :--- | :--- |
| `-f` | `--fps` | Установка частоты кадров | `15–120` (по умолчанию: 30) |
| `-n-c` | `--no-color` | Отключить цвета (ASCII-режим) | — |
| `-h` | `--help` | Вызов справочного меню | — |

### 🎨 Темы

| Флаг | Тема |
| :--- | :--- |
| `-s`, `--std` | Классический оранжевый огонь (по умолчанию) |
| `-i`, `--ice` | Ледяное пламя (голубые тона) |
| `-c`, `--classic` | Альтернативный ретро-стиль |
| `-p`, `--pink` | Розовый неоновый огонь |
| `-b`, `--blue` | Синее неоновое пламя |

### 💡 Примеры команд

```bash
# Ледяной огонь на 60 FPS
fire -i -f 60

# Синее пламя на 45 FPS
fire --blue --fps 45

# Классический ASCII-арт без цвета
fire --no-color
```
